显然模拟一个起点是否能够达到终点需要的时间复杂度，那么枚举每个点的起点就需要的复杂度，题目中虽然没有说明的规模，但我们肯定希望用更优的解法。考虑这样一个情况，假设有加油站，我们从开始枚举，遇到时汽油不够，此时停止循环，得到了，，，……，的队列，那么我们可以保证开始抵达任意一个 均有，然后考虑起始点为时到的路径，此时，那么相比于到，失去了额外油量的加持，也一定无法抵达。即若，那么必定成立。

这就意味着对于均无法完成一周的旅程，我们的下一次枚举点之间变更为，基于这样的思想，实际上只要遍历一遍数组，时间复杂度为

class Solution 
{
public:
    int canCompleteCircuit(vector<int>& gas, vector<int>& cost) 
    {
        int n = gas.size();         
        int index = 0; //起点下标
        while(index < n)
        {
            int ExtraGas = 0,i;
            for(i = 0;i < n;i++) //枚举n个点
            {
                int j = (i+index)%n; //类似循环队列
                ExtraGas += gas[j]-cost[j];
                if(ExtraGas < 0) break;
            }
            if(i == n) return index;
            else index += i+1; //getNext
        }
        return -1; //没有符合要求的点
    }
};
